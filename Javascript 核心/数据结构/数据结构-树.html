• 数据结构 - 树

  • 相关名词

    • 基础定义

      • 树结构是非线性结构
      • 有分支
      • 一个节点 关联多个节点[逻辑上]
      • 树 能够分层
      • 树 的本质上是 n 个元素的有限集合
      • 只是在逻辑上组织形式不同
      • 没有节点的树 是空树
      • 每个数都有一个根节点
      • 树的每个子集有不相关的子集构成

    • 层次

      • 规定 根节点的层次是 1

    • 树的高度 / 树的深度

      • 节点的最大层次，就是树的高度

    • 森林

      • 没有关联的 树的集合 就是森林

    • 节点

      • 节点的度 树的度

        • 节点所拥有的子树的个数，就是节点的度
        • 节点的度的最大值 就是树的度

      • 叶子节点

        • 所有节点的度===0的节点，就是叶子节点

      • 内部节点

        • 除了根几点和 叶子节点以外的节点都是内部节点

      • 父节点

        • 当前节点相关的上一层次的节点

      • 兄弟节点

        • 具有相同的父节点的节点之间，就是兄弟节点

      • 堂兄弟节点

        • 在数的同一层内的，非兄弟节点，就是 堂兄弟节点

    • 层次路径

      • 从 根节点开始 到节点 p 所经过的
      • 全部节点[包括 根节点和p节点] 叫做
      • 节点p的层次路径
      • 层次路径上的所有节点 称为 节点p的祖先节点[节点p自身除外]

    • 二叉树

      • 节点的 子节点的数量不超过 2个的树

    • 满二叉树

      • 每次的节点数量都是 满的
      • 编号规则
      • 编号从1开始
      • 从上到下 从左到右

    • 完全二叉树

      • 同满二叉树 对应编号对应位置
      • 没有跳号

    • 二叉搜索树 BST

      • 在节点的左侧保存 比父节点小的值
      • 在节点的右侧保存 比父节点大的值

    • 二叉树的储存结构

      • 顺序储存结构
        • 把二叉树的元素保存在数组中
        • index === 0 的位置 不保存元素
        • 元素从 index === 1 的位置开始保存
        • 元素的保存位置 同元素的编号相同
        • 非完全二叉树, 在索引对应的编号没有元素的情况
        • 跳过 不保存数据
      • 链式储存结构
        • 二叉树已链表的形式保存
        • 数据 说明 数据体
        • lChild [左指针域] data [数据域] rChild [右指针域]
        • { lNext: null, data: "", rNext: ""}
